fast-claude — benchmark dashboard

How to read this dashboard
  • Each row is a snapshot — a measurement you saved with benchmark.sh snapshot <name>, shown oldest → newest. Name snapshots after the change you just applied (e.g. baseline before touching anything, then incremental-hook after fixing the edit hook), so each row answers "what did that change buy?"
  • earlier snapshots  ·  latest snapshot (your current state). Shorter bar = faster/smaller = better; lower is better for every metric here.
  • The badge (▼/▲ %) compares the first snapshot to the latest: ▼ green = improved, ▲ red = regressed.
  • The top cards show the latest value of the four headline metrics, one per skill step: hook latency (Step 2), startup context (Step 3), waiting on permission prompts (Step 4), turn latency (Step 6).
  • The trend line traces the metric across all snapshots — a drop right after a snapshot means that change caused it.
